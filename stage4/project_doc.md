# 一、项目简介
##（一）需求背景
在日常工作与生活中，用户的生活服务需求呈现碎片化、场景化特征，如查询天气、计算快递费、规划出行、管理购物清单等。传统模式下，这些需求需通过不同平台或工具分别完成，操作繁琐且缺乏多轮上下文记忆能力（如用户查询 “上海天气” 后追问 “明天呢”，无法自动延续天气话题）。
为解决上述问题，本项目开发了生活助手 AI Agent，整合大模型能力与实用工具函数，通过自然语言交互，一站式解决用户各类生活需求，同时实现多轮上下文记忆，让交互更自然、更智能。
## （二）应用场景
1. 个人日常：快速查询天气、管理购物清单、核算快递费用；
2. 出行规划：结合目的地天气、行李重量、行程距离，生成个性化出行建议；
3. 办公辅助：快速核算快递费、查询日期，提升日常办公效率；
4. 学习演示：作为 AI Agent 开发实践项目，展示 “大模型 + 工具调用 + 上下文记忆” 的完整落地流程。
## （三）核心目标
1. 整合天气查询、快递费计算、出行建议、购物清单管理等核心生活服务；
2. 实现多轮上下文记忆，支持 “明天呢” 这类延续性问题的自然响应；
3. 适配命令行与 Web 端双交互方式，兼顾开发测试与实际使用需求；
4. 具备可扩展性，支持新增工具函数与复杂任务规划。
# 二、技术选型及理由
本项目采用Python 全栈技术栈，结合大模型能力与轻量级 Web 框架，兼顾开发效率、功能完整性与部署便捷性，具体选型如下：
## （一）核心开发语言：Python 3.10+
Python 语法简洁、生态丰富，拥有成熟的大模型调用、网络请求、正则解析库，完美适配 AI Agent 的开发需求；同时兼容命令行与 Web 端部署，降低跨环境开发成本。
## （二）大模型服务：通义千问（Qwen）
通义千问支持工具调用、任务规划、结果整合，适配中文自然语言理解场景，提供免费开发额度，满足项目测试与演示需求；生产环境仅需替换 API Key 即可无缝切换。
## （三）Web 后端框架：Flask
Flask 为轻量级 Web 框架，开发效率高，支持快速构建 RESTful API，适配跨域配置，完美匹配本项目 “轻量级后端服务” 的定位，无需复杂的部署流程。
## （四）关键第三方依赖
1. requests：实现高德地图 API 等网络请求，支持超时控制与异常捕获；
2. python-dotenv：实现环境变量与代码解耦，提升项目可移植性；
3. flask-cors：解决前后端跨域访问问题，实现 Web 端正常交互；
4. re/ast：实现用户输入的参数解析与工具调用适配。
## （五）开发与运行环境
• 开发环境：Windows + WSL（Linux 子系统），兼顾 Windows 的便捷性与 Linux 的部署兼容性；
• 运行环境：支持命令行交互、本地 Web 服务双模式，适配开发测试与演示场景。
# 三、核心功能说明
本项目的核心能力围绕 **“工具调用 + 上下文记忆 + 任务规划”** 展开，整合 9 大核心功能，覆盖日常生活动线，同时支持复杂任务的多工具联动。
## （一）基础工具功能
1. 实时天气查询调用高德地图天气 API，获取指定城市的实时温度、湿度、风力等信息；支持 “复用上次城市”（如用户说 “是”，自动延续上次查询城市）。
2. 快递费计算基于 “基础费 + 重量费 + 距离费” 的计费规则，根据用户输入的重量（kg）、距离（km），精准核算快递费并返回明细。
3. 日期查询支持 “今天 / 明天 / 昨天 / N 天前 / N 天后” 的日期与星期查询，适配基础时间需求。
4. 购物清单管理实现购物清单的添加、查看、移除、清空全流程管理，满足日常购物规划需求。
## （二）高级功能
1. 出行建议生成结合目的地天气、行李重量、行程距离，生成个性化出行准备建议；支持 “后天出发” 这类延续性需求。
2. 上下文记忆通过ConversationMemory类记录对话历史与核心上下文（上次查询城市、重量、距离、调用工具），支持多轮对话的自然延续。
3. 复杂任务规划将用户复杂需求（如 “帮我规划去广州的行程，带 10kg 行李，路程 1500km，后天出发，还要算 2kg 行李的快递费”）拆解为天气查询→出行建议→快递费计算的多工具调用步骤，再通过大模型整合结果。
4. 异常容错与降级针对工具调用失败、大模型请求异常、参数缺失等场景，提供友好提示；规划失败时，自动降级为大模型直接回答，保证服务可用性。
# 四、使用方法
## （一）环境准备
1. 安装 Python 3.10 及以上版本；
2. 克隆项目代码至本地，进入项目backend目录；
3. 安装依赖包：
bash
pip install dashscope flask flask-cors python-dotenv requests
## （二）环境配置
在backend目录下创建.env文件，配置以下环境变量：
##通义千问API Key（自行申请：https://dashscope.console.aliyun.com/）
DASHSCOPE_API_KEY=你的通义千问API Key
##高德地图API Key（自行申请：https://lbs.amap.com/）
AMAP_KEY=你的高德地图API Key
## （三）运行方式
本项目支持命令行交互与Web 端交互两种方式：
方式 1：命令行交互（快速测试）
进入backend目录，执行以下命令启动 Agent：
bash
python agent.py
输入指令即可交互，支持 “清空记忆”“帮助” 等特殊指令。
方式2：一键启动（推荐：start.sh）
1. 编写 start.sh 脚本
在项目根目录创建start.sh文件
2. 赋予脚本执行权限
bash
chmod +x start.sh
3. 启动服务
bash
./start.sh
4. 脚本功能说明
• 自动校验项目目录结构，避免路径错误；
• 清理残留进程，解决端口占用问题；
• 后台启动后端（Flask）和前端（http.server）服务，输出日志到backend.log/frontend.log；
• 显示服务 PID 和访问地址，提供停止服务的快捷命令。
方式3：手动启动（备用）
1. 启动后端服务：
bash
python app.py
2. 启动前端服务（新开终端）：
bash
python -m http.server 3000
3. 打开浏览器，访问http://localhost:3000，即可通过页面与 AI Agent 交互。
## （四）核心交互示例
1. 单工具交互用户：上海天气助手：🌤️ 上海：多云，22℃，湿度 65%，东风 3 级
2. 上下文延续交互用户：上海天气助手：🌤️ 上海：多云，22℃，湿度 65%，东风 3 级用户：明天呢助手：🌤️ 上海：晴，24℃，湿度 60%，东风 2 级（延续天气话题）
3. 复杂任务交互用户：帮我规划去广州的行程，带 10kg 行李，路程 1500km，后天出发，还要算 2kg 行李的快递费助手：
1. 广州后天天气：晴，26℃，湿度 55%，南风 2 级；
2. 出行建议：携带防晒霜、太阳镜，10kg 行李建议托运；
3. 2kg 行李、1500km 快递费：基础费 8 元 + 重量费 4 元 + 距离费 7.5 元，合计 19.5 元。
# 五、问题与解决方法
## （一）模块导入错误
1. 问题现象：ImportError: cannot import name 'agent_run' from 'agent'；
2. 问题原因：agent.py文件存放路径错误（误放在项目根目录，而非backend目录）；
3. 解决方法：将agent.py、m_tools.py、app.py统一放在backend目录下。
## （二）大模型调用失败
1. 问题原因：未配置 API Key 或网络异常；
2. 解决方法：检查.env文件的DASHSCOPE_API_KEY配置，确保网络可访问通义千问服务。
