# AI Agent 综合项目开发仓库

队名：海底小纵队 成员：蔡世杰 吴家秀

## 项目概述

本仓库面向机器人实验室成员，聚焦 AI Agent 核心能力落地，通过 4 个递进阶段完成「自定义 AI Agent」综合项目。项目不强制绑定特定框架，核心目标是掌握「需求分析→任务拆解→工具设计→LLM 调用→记忆 + 错误处理」的 Agent 开发全流程，最终落地一个可运行、可演示的自定义 Agent。

进阶方向：完成核心功能后，可拓展学习前后端基础内容，将 Agent 封装为简易网页/客户端，提升项目交付体验

## 阶段任务与提交要求

### 阶段 1：LLM 调用与工具设计（2.26前）

#### 核心目标

搞定 Agent 的「大脑（LLM）」和「手脚（工具）」，实现 LLM 解析意图 + 工具执行的基础链路。

#### 需完成的核心任务

1. 环境搭建
   - 完成所选框架（纯 Python/ LangChain/LlamaIndex 等）环境配置；
   - 成功调用 LLM（可调用API 也可部署本地模型），确保能正常返回响应。
2. 工具封装
   - 围绕选定选题（如学习助手 / 生活助手 / 开发助手），封装 3 个核心工具函数，要求：
     - 每个工具有明确的输入参数、输出格式、功能描述；
     - 工具可独立调用（比如「查天气工具」传入城市名能返回天气，「算题工具」传入数字能返回结果）。
3. LLM 与工具联动
   - 编写代码实现「LLM 解析用户意图 → 自动选择对应工具 → 执行工具并返回结果」的完整流程；
   - 示例：用户输入「查北京天气」→ LLM 判断意图 → 调用查天气工具 → 返回北京天气结果。

#### 提交内容

1. 选题说明：明确自己组要设计agent的功能，在本小组的README.md中介绍（接下来的阶段都将围绕此选题展开）；
2. 代码文件：`stage1/` 目录下存放环境配置说明、工具函数代码、LLM 联动代码；
3. 验证截图：`stage1/screenshots/` 目录下存放「LLM 调用成功」「工具独立运行」「LLM + 工具联动」的截图；
4. 简单总结：`stage1/README.md` 记录框架选择、工具设计思路、遇到的问题及解决方法。

#### 验收标准

- 环境可正常运行，LLM 调用无报错；
- 3 个工具封装完整，输入输出清晰；
- 实现 LLM 意图判断 → 工具选择 → 执行的基础链路。

### 阶段 2：上下文记忆与错误处理（3.1前）

#### 核心目标

让 Agent 有「记忆」（多轮交互）、能「纠错」（鲁棒性），从「能用」升级为「好用」。

#### 需完成的核心任务

1. 上下文记忆功能
   - 基于阶段 1 代码，添加对话历史存储 / 管理逻辑（列表 / 字典 / 框架自带 Memory 模块）；
   - 实现多轮交互：Agent 能记住上一轮对话内容（比如「查北京天气」→ 「明天呢」→ Agent 自动查询北京明天天气）。
2. 错误处理功能
   - 针对 3 个核心工具，添加至少 3 种常见错误处理（try/except）：
     - 示例：查天气未输入城市、算题输入非数字、LLM 输出格式错误；
   - 错误发生时 Agent 不崩溃，且能给出清晰提示（比如「请输入正确的城市名」）。
3. Prompt 优化
   - 优化 LLM Prompt，让 Agent 能主动追问缺失参数（比如用户输入「保存笔记」→ Agent 问「请输入笔记内容」）；
   - 能解析模糊指令（比如「算一下钱」→ Agent 追问「请告诉我要计算的金额和费率」）。

#### 提交内容

1. 代码文件：`stage2/` 目录下存放新增记忆、错误处理、Prompt 优化后的完整代码；
2. 测试用例：`stage2/test_cases.md` 记录 5-8 种测试用例（正常输入 / 错误输入 / 模糊输入）及对应输出；
3. 验证截图：`stage2/screenshots/` 目录下存放「多轮交互」「错误处理」「主动追问」的截图。

#### 验收标准

- 多轮交互正常，Agent 能记住历史对话；
- 能处理 3 种以上错误场景，提示清晰不崩溃；
- 能主动追问缺失参数，解析模糊指令。

### 阶段 3：任务规划与多工具协作（3.4前）

#### 核心目标

让 Agent 能拆解复杂任务、联动多个工具完成目标，具备核心竞争力。

#### 需完成的核心任务

1. 任务规划实现
   - 编写逻辑让 LLM 将复杂需求拆解为「工具调用步骤清单」；
   - 示例：用户输入「生成今日学习周报」→ LLM 拆解为「读取笔记 → 整理内容 → 保存周报」。
2. 多工具协作
   - 实现 Agent 按拆解步骤，依次调用多个工具并整合结果；
   - 完成至少 2 个复杂任务场景的多工具联动（贴合自身选题）；
   - 示例：学习助手 → 「查单词 AI → 翻译单词 → 保存到错题本」。
3. 结果整理优化
   - Agent 能将多个工具的输出结果，整理为连贯的自然语言返回（而非分条输出）；
   - 示例：多工具联动后返回「上海今日 22℃，快递时效 2 天，快递费 10 元」。

#### 提交内容

1. 代码文件：`stage3/` 目录下存放任务规划、多工具协作的完整代码（可新增 1 个工具）；
2. 步骤清单：`stage3/plan_steps.md` 记录 2 个复杂任务的拆解步骤；
3. 验证截图：`stage3/screenshots/` 目录下存放「任务拆解」「多工具联动」「结果整理」的截图。

#### 验收标准

- 能拆解复杂需求，输出清晰的工具调用步骤；
- 实现 2 个以上多工具协作场景，结果整理自然；
- 新增 1 个工具，完善项目功能。

### 阶段 4：落地交付 - 综合项目整合与文档编写（3.8前）

#### 核心目标

整合所有知识点，落地完整的自定义 Agent，完成需求分析、测试优化、文档编写。

#### 需完成的核心任务

1. 项目整合
   - 整合前 3 阶段代码，完善所有功能，实现一个可交互的完整 Agent（命令行 / 前后端均可）；
   - 全面测试：设计 10+ 测试用例（正常 / 错误 / 模糊 / 复杂需求），修正所有 bug。
2. 文档编写
   - 完成项目文档 stage4/project_doc.md，包含：
     - 项目简介（需求、应用场景）；
     - 技术选型（框架选择及理由）；
     - 核心功能说明；
     - 使用方法（运行步骤、交互方式）；
     - 遇到的问题及解决方法；
     - 未来可优化方向。
3. 演示准备
   - 录制 1-3 分钟演示视频，展示 Agent 核心功能。

#### 提交内容

1. 最终代码：`stage4/final_code/` 目录下存放完整可运行代码（含依赖说明 `requirements.txt`）；
2. 项目文档：`stage4/project_doc.md`；
3. 测试用例：`stage4/full_test_cases.md`（10+ 个测试用例）；
4. 演示材料：`stage4/demo/` 目录下存放演示视频 / 截图。

#### 验收标准

- 代码完整可运行，无明显 bug；
- 文档内容完整，逻辑清晰；
- 演示材料能覆盖所有核心功能。

## 选题方向（任选其一，也可自定义）

1. **学习助手**：查单词、算题、保存错题、生成学习计划、整理笔记；
2. **生活助手**：查天气、算快递费、记录购物清单、查询节假日、简单翻译；
3. **开发助手**：生成简单 Python 代码、检查语法错误、保存代码文件、查询基础语法；
4. **文档助手**：读取本地文档、总结文档内容、提取关键信息、生成文档摘要。

## 注意事项

1. 优先保证 Agent 核心功能，极简前后端为可选进阶项（基础弱的同学可保留命令行交互）；
2. 代码需添加清晰注释，确保可读性；
3. 文件夹名字为组名，在README.md文件中首先说明小组成员；
4. 提交内容需按阶段目录存放，命名规范，便于检查。

## 验收流程

1. 每个阶段截止后，组长提交该阶段代码 / 文档 / 截图至对应目录；
2. 最终阶段提交后，进行项目演示，讲解核心功能及技术要点；
3. 验收标准分「基础档（及格）」和「进阶档（优秀）」，兼顾不同基础同学。
